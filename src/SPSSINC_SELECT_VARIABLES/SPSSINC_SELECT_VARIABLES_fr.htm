Définition d'une macro répertoriant les variables sélectionnées

Cette commande permet de définir une macro SPSS dont la valeur est une liste de valeurs répondant aux critères, tels que le niveau de mesure, le type, les attributs de variable et les motifs dans les noms.

Grâce à cette approche, vous pouvez créer des travaux de syntaxe généralisés qui n'ont pas besoin du nom exact d'une variable dans un fichier de données particulier. Par exemple, vous pouvez créer un travail qui affiche les statistiques descriptives appropriées pour les variables d'un jeu de données en fonction de leur niveau de mesure.

Nom de macro : Entrez un nom pour la macro à générer. Il est admis de commencer chaque nom par "!" afin d'éviter tout conflit de nom avec les autres variables.

Variables et Limite de sélection : Déplacez les variables vers le contrôle de limite de sélection afin de réduire le nombre de résultats à un sous-ensemble de ces noms uniquement. Si aucun autre critère n'est défini, la macro doit désigner exactement ces noms.

•Si le contrôle de limite est vide, les autres critères sont appliqués à l'ensemble des variables.

Type de variable : Limitez la sélection aux variables numériques, aux variables de chaîne ou aux deux (aucune limite).

Niveau de variable : Sélectionnez le critère de niveau de mesure de variable à appliquer.

Rôle de variable : Dans Statistics version 18 ou ultérieure, vous pouvez affecter des rôles aux variables. Sélectionnez le critère de rôle à appliquer. Dans la version 17, ce critère est ignoré.

•Le rôle est particulièrement utile si cette boîte de dialogue est utilisée dans le modélisateur.

Expression régulière de nom de variable : Vous pouvez saisir une expression régulière pour filtrer les variables en fonction des noms. Les expressions régulières décrivent un motif de caractères. Ce motif est sensible à la casse et commence au début du nom. Exemples :

age - toutes les variables commençant par age

.*age - toutes les variables dont le nom contient le mot age

.*\d$ - toutes les variables dont le nom se termine par un chiffre

Attributs

La sous-boîte de dialogue Attributs vous permet de spécifier des critères en fonction d'attributs personnalisés. Les attributs personnalisés sont des propriétés de variables pouvant être créées par la commande VARIABLE ATTRIBUTE, par la boîte de dialogue Définir les propriétés de variable ou dans l'éditeur de données. Ils sont enregistrés avec le jeu de données.

Les variables doivent avoir tous ces attributs : Les variables sélectionnées doivent avoir tous les attributs répertoriés ici. Entrez un ou plusieurs noms d'attribut séparés par un espace. La valeur de l'attribut, si elle existe, n'a pas d'importance.

Les variables doivent avoir au moins un de ces attributs : Les variables répondent à ce critère lorsqu'elles possèdent au moins l'un des attributs de la liste. Entrez un ou plusieurs noms d'attribut séparés par un espace. La valeur de l'attribut, si elle existe, n'a pas d'importance. Les noms ne sont pas sensibles à la casse.

Valeurs d'attribut : Entrez des noms d'attributs et des listes de valeurs pour restreindre les variables sélectionnées à celles qui possèdent cette valeur. Vous pouvez entrer jusqu'à trois ensembles de noms et de valeurs.

•Chaque valeur doit être placée entre apostrophes ou guillemets et s'il y a plusieurs valeurs, celles-ci doivent être séparées par un espace.

•Les valeurs sont sensibles à la casse : Si un attribut de variable appelé unités possède la valeur EUROS, vous devez saisir "EUROS" dans le champ de valeur des unités afin de sélectionner la variable.

•Si la variable ne possède pas d'attribut spécifique, le test est ignoré. Indiquez l'attribut dans le contrôle Les variables doivent avoir tous ces attributs pour exclure les variables qui ne possède pas cet attribut.

•Si l'attribut est un attribut de tableau, dans lequel plusieurs valeurs peuvent exister pour un même attribut, la variable répond aux critères si une valeur du tableau correspond à une valeur de la liste Valeurs des attributs.

Options

Ordre des variables sélectionnées dans la macro : Vous pouvez indiquer que la liste de variables définie par la macro est classée par ordre alphabétique ou par ordre d'apparition des variables dans le fichier. Si l'option "Ordre de la liste" est sélectionnée, toutes les variables de la liste Limite de sélection qui répondent aux autres conditions apparaissent dans l'ordre indiqué dans la définition de macro.

Si aucune variable n'est sélectionnée : Si aucune variable ne répond aux critères de sélection, vous pouvez soit générer une erreur et ne pas définir la macro, soit poursuivre et générer une macro vide. La macro vide peut être testée ultérieurement à l'aide du code de macro afin d'éviter la génération d'erreur due à l'absence de valeur.

Imprimer la définition de macro : Choisissez si le contenu de la macro est répertorié dans un tableau croisé dynamique ou non.

Caractère de séparation : Indiquez si vous voulez séparer les noms de variables par un espace ou un signe plus dans la définition de macro. Le signe plus se révèle particulièrement utile pour les expressions TABLE des tableaux personnalisés.

Fonctions supplémentaires

Exécutez la commande
SPSSINC SELECT VARIABLES /HELP.
pour afficher la description intégrale de la syntaxe.

Un dictionnaire des macros définies par cette commande est mis à la disposition des programmeurs en tant que fichier SPSSINC_SELECT_VARIABLES.macrodict.

Vous pouvez spécifier une liste des rôles.

Vous pouvez utiliser TO et ALL dans la syntaxe avec des variables spécifiques.

Conditions requises

Cette commande requiert le plug-in de programmabilité Python, fourni sur le CD-ROM IBM SPSS Statistics, ou téléchargeable à partir de SPSS Developer Central (www.spss.com/devcentral).
